05 Part 1：完整演示——从创建到部署

🎯 建议直接跟做！下列命令在终端完成「创建 Next.js → 推到 GitHub → 部署到 Vercel」。

没有 gh 时，可以安装，也可在 GitHub 网页新建空仓库，再 git remote add + git push

1

创建 Next.js 项目

在终端创建一个新的 Next.js 应用

▼
终端 
# 创建 Next.js 项目（选择默认配置一路回车）
npx create-next-app@latest my-project

# 进入项目目录
cd my-project

# 启动开发服务器预览
npm run dev
💡 创建时确认：TypeScript ✓、ESLint ✓、Tailwind ✓、App Router ✓
2

初始化 Git 并提交

把项目纳入版本控制

▼
终端 
# 初始化 Git 仓库
git init

# 暂存所有文件
git add .

# 首次提交
git commit -m "feat: initial commit with Next.js"
3

用 GitHub CLI（gh）创建远程仓库并推送

前提：已安装 gh 并完成 gh auth login

▼
终端 
# 创建 GitHub 仓库并推送（-y 确认默认选项）
gh repo create my-project --public --push --source=.

# 或分步操作：
gh repo create my-project # 创建空仓库
git remote add origin https://github.com/YOUR_USERNAME/my-project.git
git branch -M main
git push -u origin main
⚠️ 注意：首次使用需先执行 gh auth login 进行浏览器授权
4

安装并登录 Vercel CLI

通过命令行连接 Vercel 账号

▼
终端 
# 安装 Vercel CLI
npm i -g vercel

# 登录（会打开浏览器）
vercel login
5

部署到 Vercel

把项目部署到 Vercel 平台

▼
终端 
# 部署到 Vercel（按提示操作）
vercel

# 生产环境部署
vercel --prod
🎉 完成！Vercel 会返回生产环境 URL，如 https://my-project.vercel.app

06 Part 2：自动化部署管道与 CI/CD

🔄

什么是 CI/CD？

CI = Continuous Integration（持续集成）
CD = Continuous Delivery/Deploy（持续交付/部署）
代码提交后自动完成构建、测试、部署。

🗺️

和 Cursor + GitHub + Vercel 的对应关系

CI → GitHub 监听 push 事件，触发构建
CD → Vercel 自动构建并部署到 CDN
你的角色：只管写代码和 push。

❓ 常见问题排查

Q 部署失败怎么办？
检查 Vercel Dashboard 的 Deployments 页面，查看构建日志。常见问题：依赖安装失败、环境变量缺失、构建命令错误。
Q 如何回滚到之前的版本？
在 Vercel Dashboard → Deployments → 选择之前的部署 → 点击 "..." → Promote to Production。
Q 如何查看生产环境 URL？
运行 vercel --prod 后终端会显示 URL，或在 Vercel Dashboard 的 Domains 处查看。
